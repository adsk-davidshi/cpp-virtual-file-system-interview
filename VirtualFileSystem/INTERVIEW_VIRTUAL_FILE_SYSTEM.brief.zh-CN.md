# 虚拟文件系统面试题

> 仓库首页会直接渲染根目录 `README.md`。
> 本文档与根目录 `README.md` 保持同样内容，便于在当前目录下继续查看和编辑。

## 1. 背景

本仓库包含一个使用 C++14 实现的简化版内存虚拟文件系统。
现有代码和单元测试都不完整，部分方法只是骨架实现。

核心类型包括：

- `IVirtualNode`
- `VirtualFolder`
- `VirtualFile`
- `VirtualRootFolder`
- `VirtualFileSystem`

根目录使用 `$` 表示，路径分隔符使用 `/`。

---

## 2. 任务

请阅读现有代码和测试，在当前结构基础上补齐缺失功能，并让测试约束这些行为。

本题重点不是重写，而是：

- 理解现有设计
- 在必要处做小范围重构
- 完成功能
- 补充测试

---

## 3. 约束

- 使用现有 C++14 工程和现有 MSTest 测试工程
- 尽量复用当前类结构，不要整体推翻重写
- 当前 starter code 已经固定了本题会用到的接口形状，包括 `GetFile(...)`、`Rename(...)`、`Remove(path)`、`ListChildren()`
- 允许调整不安全的返回类型或内部存储结构，但需要在设计说明中解释原因
- 所有新增行为都必须通过测试约束

---

## 4. 交付物

提交内容至少包括：

1. 完整实现
2. 通过后的单元测试
3. 你新增的测试
4. 简短设计说明 `DESIGN_NOTES.md`

设计说明至少说明：

- 所有权策略
- 是否修改了现有接口
- 路径校验如何实现
- 你如何处理删除、重命名、遍历和并发读取

## 5. 具体要求

下面的要求按难度分层。

### 5.1 难度1

补齐并修正最基础的路径与目录能力：

- `CreateFolder("A")`
- `CreateFolder("A/B")`
- `GetFolder("A/B")`
- `getPath()`

要求：

- 根目录始终为 `$`
- 重复创建同一目录返回已有对象
- `CreateFolder("$")` 和 `CreateFolder("")` 必须失败
- 至少拒绝一种非法路径，例如 `"A//B"`

### 5.2 难度2

补齐文件能力，并让基础测试完整：

- 增加 `GetFile(...)`
- 支持 `CreateFile("a.txt")`
- 支持 `CreateFile("A/B.txt")`

要求：

- 若中间目录不存在，应自动创建
- 重复创建同一文件返回已有对象
- 文件与目录同名冲突时，行为必须明确且可测

### 5.3 难度3

实现删除与重命名。

至少包括：

- 节点删除
- `Remove(path)`
- `Rename(...)`

要求：

- `Remove(path)` 至少需要能删除文件
- 根目录禁止重命名
- 文件夹重命名后，其后代节点路径必须正确更新
- 删除或重命名后，原父子关系和查询结果必须保持一致

### 5.4 难度4

增加遍历接口，并支持基本并发读取。

至少包括：

- `ListChildren()` 或等价接口
- 多线程并发读取

要求：

- 返回直接子节点
- 同时包含文件和文件夹
- 遍历顺序必须确定，建议按名称排序
- 在基本并发读取场景下，不能出现崩溃或明显不一致结果

### 5.5 难度5

现场讨论或继续实现以下任一方向：

- `copyTo(...)` / `moveTo(...)`
- 更完整的线程安全方案
- 异常安全与失败回滚

如果你选择实现 `copyTo(...)` / `moveTo(...)`，至少要说明：

- 目标节点类型如何校验
- 名称冲突如何处理
- 为什么不会把文件夹移动到自己的后代目录下

如果你选择讨论线程安全或异常安全，至少要说明：

- 你的锁粒度选择
- 哪些接口属于读，哪些属于写
- 失败时如何避免留下半完成状态

## 6. 最低测试要求

你至少需要补充并通过以下测试：

- 禁止创建名为 `$` 的子目录
- 禁止创建空目录名
- 创建嵌套目录
- 重复创建同一目录返回同一对象
- 目录完整路径正确
- 可以按路径查询目录
- 至少一种非法路径会被拒绝
- 创建嵌套文件
- 文件与目录同名冲突
- `Remove(path)` 删除文件
- 重命名文件夹后路径更新
- 遍历结果顺序稳定
- 至少 1 个基本并发测试

如果面试官事先补充了失败测试，请先让这些测试通过。

- 是否做了合理而克制的重构
- 是否正确处理路径、删除、重命名和所有权
- 是否能写出真正约束行为的测试
